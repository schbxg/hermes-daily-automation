# Growth playbook (for maintainers)

Star growth for a **template pack** is mostly distribution + trust, not more YAML.

## After every release

1. [ ] Tag GitHub Release (`v0.2.0`, …) with CHANGELOG bullets  
2. [ ] Post in **Hermes Discord** / Discussions with: 5-min demo, 1 sample output, cost line  
3. [ ] X / 即刻 / V2EX: “数字员工第 N 天” series, not “请 star”  
4. [ ] PR real (redacted) Telegram screenshots into `screenshots/`  
5. [ ] Offer upstream cookbook / community list link if Hermes accepts community packs  

## Positioning one-liners

- EN: *Personal AI morning OS for Hermes — Progressive Cron so agents don't re-teach Day 1.*  
- ZH: *Hermes 个人晨间 OS：Progressive Cron 让定时任务按天递进、不重复。*

## What not to do

- Don't lead with Star History at 1⭐  
- Don't claim “official” Hermes unless upstream links you  
- Don't ship personal absolute paths or API keys in examples
