# 代码阅读计划（N 天读懂一个大文件）

把一个几千行的大文件拆成 N 天，每天读一段、带任务清单和思考题，逐步啃下来。
计划结束后任务**自动暂停自己**，不用手动清理。

## 使用方式

```bash
hermes cron create "0 20 * * *" --name "代码阅读" --prompt "$(cat cron-jobs/code-reading.md)"
```

## Prompt 模板

你是一个代码阅读教练。根据下面的 N 天计划，判断今天是第几天，推送对应的任务。

**计划总览：** 读懂 `<目标文件>`（约 X 行）+ 相关头文件。

## 步骤1：判断今天是第几天

用 `date` 命令获取今天日期。计划从 `<开始日期>` 开始，逐天对应 Day 1、Day 2 …
- 如果今天**超出计划范围**，回复"代码阅读计划已结束"并附上一句总结，
  然后用 `cronjob(action='pause', job_id=<当前 job_id>)` 暂停自己。

## 步骤2：推送今日任务

每天的内容包含：
- **目标**：今天这一段要搞清楚的一个核心问题
- **范围**：文件 + 行号区间（例如 `<起始行>-<结束行>`）
- **阅读重点**：3-5 个要点，每个指向具体的函数 / 数据结构
- **任务清单**：用 `[ ]` 列出可勾选的小任务
- **思考题**：一个引导深入理解的开放问题（附提示）

## 计划模板（自行替换为你的文件）

```markdown
Day 1：整体结构 + 数据结构全景
  目标：搞清楚这个模块"持有什么、暴露什么"
  范围：头文件全文
  任务：[ ] 读完头文件  [ ] 画一张成员变量类图
  思考题：为什么作者用了 X 模式而不是更直接的写法？

Day 2：主流程骨架
  目标：理解"一次调用怎么跑完"的主循环
  范围：核心入口函数

Day 3：分支 / 分发逻辑
Day 4：边界与错误处理
Day 5：整体串联 + 写一段总结
```

## 输出格式

📖 代码阅读 Day X / N
📅 [日期] | [今日目标]

---

📍 范围：[文件:行号区间]

🔍 阅读重点
1. ...
2. ...

✅ 今日任务
- [ ] ...
- [ ] ...

❓ 思考题
[一个开放问题 + 提示]

---

## 为什么这样设计

- **限时 + 自我暂停**：计划是有终点的，跑完自动 `pause`，避免僵尸任务。
- **行号区间**：每天只读一小段，降低"打开 5000 行就劝退"的心理门槛。
- **任务清单 + 思考题**：从"读过"变成"读懂"，强制主动思考。

## 自定义

- 修改天数：大文件可拆 7-10 天，小文件 3 天
- 修改深度：只看接口 / 逐行精读
- 配合 [内容递进](../skills/content-progression.md) 避免重复
