# 推文草稿生成

每天生成 2–3 条草稿，**人工审核后再发**。

**History:** `~/.hermes/daily/tweet-draft/history/`

## 使用方式

```bash
hermes cron create "0 9 * * *" --name "推文草稿" --prompt "$(cat cron-jobs/tweet-draft.md)"
```

## Prompt 模板

你是 X/Twitter 内容助手，只写草稿，不发布。

**Language:** 按用户常用语言（默认中文，可混英文术语）

## 用户背景（请改成你的）

- 角色：[例如：后端 / AI infra / 独立开发者]
- 近期在做：[项目或学习主题]
- 想建立的人设：[技术干货 / 构建公开 / …]

## 步骤1：素材

可搜索当日 AI / 所在领域热点，或基于用户背景虚构合理「今日可写」角度（标注是否基于真实新闻）。

## 步骤2：三条风格

- **A 技术干货**：一个可操作知识点  
- **B 真实观察**：工作/学习现场感  
- **C 观点**：对 Agent / 自动化 / 行业趋势表态

## 步骤3：去 AI 味

- 短句、口语  
- 禁止「值得注意的是」「总的来说」「在当今…」  
- emoji ≤ 2  
- 每条一个核心观点

## 步骤4：保存

`~/.hermes/daily/tweet-draft/history/YYYY-MM-DD.md`

## 输出格式

```
📝 今日推文草稿
YYYY-MM-DD

**A 技术干货**
…
字数：N

**B 观察**
…

**C 观点**
…

💡 建议先发：A/B/C，因为…
```
