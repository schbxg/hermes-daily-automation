# 内容递进 Skill

防止 Cron Job 每天推送重复内容的通用方案。

## 核心思路

用**文件系统做 Agent 的外部记忆**：

```
1. 学习计划文件（静态）  → 告诉 Agent "今天该讲什么"
2. 历史记录目录（动态）  → 告诉 Agent "之前讲过什么"
3. 日期计算（锚点）      → 告诉 Agent "今天是第几天"
```

三者结合，保证内容每天递进不重复。

## 实现模板

在你的 Cron Job prompt 中加入以下步骤：

```markdown
## 步骤1：确定今天是第几天
运行 `date +%Y-%m-%d` 获取今天日期。
计算从你的开始日期（Day 1）算起是第几天。

## 步骤2：读取学习计划
读取 `/path/to/plan.md` 文件，找到今天对应的主题。

## 步骤3：读取历史记录
运行 `ls /path/to/history/` 查看已推送的内容。
读取最近 3-5 天的文件，确保今天的内容与之前不同。

## 步骤4：生成内容
根据今天的主题，生成内容。
确保与历史记录中的内容不重复。

## 步骤5：保存到历史
将今天的内容保存到 `/path/to/history/day_XX_主题.md`
```

## 目录结构

```
your-project/
├── plan.md                    # 学习计划（静态）
├── history/                   # 历史记录（动态）
│   ├── day_01_主题A.md
│   ├── day_02_主题B.md
│   └── ...
└── AGENTS.md                  # Agent 指南
```

## 学习计划模板

```markdown
# 学习计划

> Day 1 = 2026-05-22

| Day | 主题 | 关键词 |
|-----|------|--------|
| 1 | 基础概念A | 关键词1 |
| 2 | 基础概念B | 关键词2 |
| 3 | 进阶概念A | 关键词3 |
| ... | ... | ... |
```

## 领域轮换模板

如果不想按固定计划，可以用 mod 轮换：

```markdown
计算今天是第几天，按 mod N 轮换领域：
- Day mod 7 = 1: 领域A
- Day mod 7 = 2: 领域B
- Day mod 7 = 3: 领域C
- ...
- Day mod 7 = 0: 领域G
```

## 常见问题

### Q: Agent 不会读历史文件怎么办？
A: 在 prompt 中明确写"运行 `ls` 和 `read_file` 查看历史"。

### Q: 历史文件太多怎么办？
A: 只读最近 3-5 天，不需要读全部。

### Q: 想跳过某天的内容怎么办？
A: 手动在历史目录创建一个占位文件，Agent 会自动跳过。

### Q: 想重置进度怎么办？
A: 清空历史目录，Agent 会从 Day 1 重新开始。
