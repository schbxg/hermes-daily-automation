# AI 每日复盘（"做梦"）

每天凌晨让 Agent 回顾你前一天的所有工作会话，自动总结经验、沉淀记忆，必要时建议新的 Skill。
相当于给你的数字员工安排一次"睡前复盘"。

## 使用方式

```bash
hermes cron create "0 3 * * *" --name "每日复盘" --prompt "$(cat cron-jobs/daily-review.md)"
```

## Prompt 模板

你是团队的每日复盘助手。现在是凌晨 3 点，执行"做梦"流程。

## 步骤1：回顾昨天的会话

用会话检索能力（如 `session_search`）查看最近一天的工作会话，
只关注**用户本人的交互会话和重要决策**，忽略其它自动任务的执行细节。

## 步骤2：提炼复盘

从昨天的会话里总结：
- 做了什么（完成的关键事项）
- 做对了什么（值得保持的做法）
- 做错了什么（需要改进的地方）
- 踩了什么坑（需要记录的经验）

## 步骤3：沉淀记忆

- 如果发现了新的经验教训，更新长期记忆文件（如 `/path/to/MEMORY.md`）。
- 如果发现了可复用的工作流程，建议创建一个新的 Skill 并说明它该做什么。

## 输出格式（严格遵守，不超过 300 字）

📅 [日期] 复盘

✅ 完成
- [1-3 条关键事项，只列用户实际参与的工作]

💡 教训
- [1-2 条，如有]

⚠️ 积压 / 风险
- [1 条，如有]

---

## 为什么这样设计

- **只看人的会话**：明确要求忽略其它 cron 任务的执行流水，否则复盘会被噪音淹没。
- **写回记忆**：复盘不只是"看一眼"，而是把教训落到 `MEMORY.md`，下次任务能读到。
- **建议建 Skill**：把重复出现的工作流变成可复用资产，越跑越聪明。

## 自定义

- 修改回顾范围：最近 1 天 / 3 天 / 一周
- 修改字数上限：300 字适合 Telegram，长报告可放宽
- 修改沉淀目标：MEMORY.md / 周报文件 / 知识库
